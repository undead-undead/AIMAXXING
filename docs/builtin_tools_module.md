# AIMAXXING Built-in Tools Module Documentation

The `builtin-tools` crate contains the standardized library of tools (skills) available to AIMAXXING agents. It provides a wide range of capabilities, from filesystem manipulation to web browsing and chart generation.

## 🛠 Technology Stack & Dependencies

- **Web Browing**: `headless_chrome` for Chromium-based automation.
- **Data Visualization**: `plotters` and `image` for chart rendering.
- **Code Execution**: Integrated with the `runtimes` crate for JS/Python interpretation.
- **Serialization**: `serde_json` and `schemars` for automated argument schema generation.
- **Communication**: `lettre` for SMTP and `notify-rust` for system notifications.
- **Git Integration**: Wraps the local `git` binary for repository operations.

## 📂 Architecture & Modules

### 1. Functional Tools
- **`filesystem.rs`**: Safe, sandboxed operations for reading, writing, and editing files.
- **`browser.rs`**: Headless browser control for complex web interactions.
- **`web_search.rs` / `web_fetch.rs`**: Lightweight search and scraping utilities.
- **`chart.rs`**: Data visualization and image generation.

### 2. Meta-Tools & Coordination
- **`code_interpreter.rs`**: Safely executes dynamic code generated by the LLM.
- **`forge.rs`**: Allows agents to compile new Rust tools at runtime.
- **`delegation.rs` / `handover.rs`**: Logic for swarm coordination and human-in-the-loop triggers.

### 3. Media & Processing
- **`voice.rs`**: Interface for speech-to-text (Whisper) and text-to-speech.
- **`image.rs`**: Pixel manipulation and vision model preparation.
- **`text_extract.rs`**: Multi-format document parsing (PDF, DOCX).

## 🚀 Purpose

The `builtin-tools` module's purpose is to **Empower the Agent**. By providing a rich, "batteries-included" set of native Rust tools, AIMAXXING ensures that agents can perform complex real-world tasks immediately upon deployment without requiring custom integration code for common operations.
